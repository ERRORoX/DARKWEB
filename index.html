Боти Telegram + панели админ

Мағозаи ноутбук дар Telegram

Боти комил барои каталог, фармоишҳо, пардохт ва машваратдиҳии AI. Веб-панели админ барои идоракунӣ. Минтақа: Тоҷикистон (забонҳои тоҷикӣ ва русӣ).

  • Каталог, фармоишҳо, реквизитҳо, чекҳо, статусҳо
  • Интихоби ноутбук тавассути AI, зиёда забонӣ
  • Ҳуҷҷатҳо, Docker, супориши ба фармоишгар
Тавсифи лоиҳа
B2C e-commerce

Чи гуна система

  • Навъ: Боти Telegram мағозаи интернетӣ (ноутбук) + веб-панели админ.
  • Мизоҷон: харидор бо бот дар Telegram алоқа дорад — каталог, фармоиш, реквизитҳо, чек.
  • Админ: шумо фармоишҳо ва маҳсулотро тавассути браузер идора мекунед (дашборд, филтрҳо, содирот, танзимот).
  • Минтақа: Тоҷикистон; забонҳо — тоҷикӣ ва русӣ дар бот ва дар танзимот.
  • Ҳаҷм: як мағоза, як бот, як панели админ; додаҳо назди шумо нигоҳ дошта мешаванд (SQLite, Docker).
Вазифаҳо
Бот + админ

Бот барои мизоҷон

  • Каталог бо категорияҳо (бозӣ, таҳсил, кор), акс/видео, ҷустуҷӯ, дӯстдошта.
  • Тартиб додани фармоиш: НАМ, телефон, шаҳр, суроға; рақами худкор.
  • Реквизитҳои пардохт, боргузории акси чек, таърихи фармоишҳо, статусҳо.
  • Машваратдиҳии AI: интихоб аз рӯи буҷа ва талабот, тоҷикӣ/русӣ/англисӣ.
  • Саволу ҷавоб, контактҳои дастгирӣ, ёддошт дар бораи чек (ҳар 6 соат).

Веб-панели админ

  • Даромад бо логин ва калиди махфӣ; чанд администратор.
  • Фармоишҳо: филтрҳо, тағйири статус, дидани чек, содироти CSV.
  • Маҳсулот: CRUD, акс/видео, боқимонда, намуди ҷадвал/плитка.
  • Дашборд, огоҳиҳо (фармоишҳои нав, боқимондаи паст), танзимоти .env, нусхаи БД.
Технологияҳо
Стек
  • Backend: Python 3.11+, aiogram 3.x, aiohttp, aiosqlite, python-dotenv.
  • БД: SQLite 3 (дастрасии асинхронӣ).
  • Панели админ: HTML5, CSS модулӣ, Vanilla JavaScript (ES-модулҳо).
  • API: Telegram Bot API, OpenRouter API (AI).
  • Бехатарии: JWT, SHA256, филтри IP ихтиёрӣ.
  • Сарпуниш: Docker, docker-compose; дастур барои Windows, Linux, macOS.
Чи дар нарх дохил аст
Супориш
  • Коди ибтидоии бот ва панели админ (дастрасии комил).
  • Пойгоҳи додаҳои омода (схема, миграсияҳо дар зарурат).
  • Тасвири Docker ва docker-compose барои оғоз дар компютери шумо ё сервер.
  • Дастури насби барои фармоишгар (Windows + Docker Desktop, қадам-ба-қадам).
  • Дастури барои барномасоз (соз, Docker Hub, такмил).
  • Ҳуҷҷатҳо: тавсифи лоиҳа, тавсиф барои арзёбӣ, README.
  • Супориши мавод: архиви лоиҳа ва/ё тасвир дар Docker Hub + дастурҳо.
  • Як машварат дар бораи оғоз (ҷавоб ба саволҳо то 7 рӯз пас аз супориш).

Супориши маҳсулот ба фармоишгар
Ду вариант

Варианти 1 — архив бо лоиҳа

  • Архив (ZIP) бо коди ибтидоии комил, Dockerfile, docker-compose.yml, .env.example.
  • Дастури «Насб барои фармоишгар» (Windows, Docker Desktop, қадам-ба-қадам).
  • Шумо распак мекунед, .env эҷод мекунед (токени бот, сирри админ), оғоз: docker compose up -d. Панели админ: http://localhost:8080.

Варианти 2 — тасвири Docker Hub

  • Тасвир дар Docker Hub нашр мешавад. Номи тасвир + дастур ба шумо супорида мешавад.
  • Шумо ҷузвдор эҷод мекунед, файли .env ва як фармон иҷро мекунед: docker run ё docker compose -f docker-compose.hub.yml up -d.
  • Коди ибтидоӣ супорида намешавад (фақат тасвири омода барои оғоз).
Пардохт
Шартҳо
  • Асъор: фақат сомонӣ (TJS) — Тоҷикистон.
  • Усул: гузарониш ба ҳисоб/корт (реквизитҳо ҷудо мувофиқа мешаванд) ё платформаи пардохтӣ (масалан, PayPal) бо розигӣ.
  • Марҳалаҳо (схемаи тавсияшуда):
  • 30% — пешакӣ пас аз имзои шартнома / тасдиқи ТЗ.
  • 40% — пас аз супориши нусхаи корӣ (бот + админ, санҷиш аз тарафи шумо).
  • 30% — пас аз супориши ҳамаи мавод (код/тасвир, дастурҳо, машварат) ва имзои акт.
  • Тақсимоти дигар бо дархости шумо имконпазир аст (масалан, 50/50).
Шартнома
Шартҳои ҳамкорӣ
  • Мавзӯъ: таҳия ва супориши боти Telegram мағозаи интернетӣ ва веб-панели админ мувофиқи тавсифи мувофиқашуда.
  • Мӯҳлат: ҷудо мувофиқа мешавад (ориентир: 2,5–4 моҳ дар ҳолати корои қисмӣ).
  • Ҳуқуқ: пас аз пардохти комил фармоишгар ҳуқуқи истифодаи код/тасвири супоришашударо барои мақсадҳои худ (шумора тиҷорӣ) мегирад. Коди ибтидоӣ дар интихоби варианти архив супорида мешавад.
  • Кафолат: рафъи хатоҳои муҳим (номеҳнатӣи функсионалитети тавсифшуда) то 14 рӯз пас аз супориш дар шарти риояи дастури насб.
  • Махфият ва ТЗ: шартҳо дар шартнома равшан мешаванд (имзо ба тариқи электронӣ ё рӯи коғаз — бо розигӣ).
Талаботи аз тарафи шумо
Фармоишгар
  • Пардохти бавақт бо марҳала (реквизитҳо ва мӯҳлатҳо мувофиқа мешаванд).
  • Эҷоди боти Telegram тавассути @BotFather ва супориши токен ба барномасоз (ё гузоштани худ дар .env мувофиқи дастур).
  • Компютери бо Windows 10/11 (64-bit) ва насбшудаи Docker Desktop — барои оғоз мувофиқи дастур (ё Linux/macOS дар оромӣ).
  • Оқибатдиҳӣ дар мӯҳлатҳои қарорёфта (қабули марҳалаҳо, ҷавоб ба саволҳо оид ба санҷиш — то 5–7 рӯзи корӣ).
  • Додани реквизитҳои пардохт барои мизоҷон, контактҳои дастгирӣ ва матнҳо (ё розигӣ ба истифодаи намуна аз лоиҳа).
  • То қабул тағйирот дар код ворид накардан (ё такмилҳоро ҷудо мувофиқа кардан).
Шумо аз ман чӣ мегиред
Натиҷаи супориш
  • Боти Telegram коркунанда бо каталог, фармоишҳо, реквизитҳо, боргузории чекҳо, машваратдиҳии AI ва дастгирии тоҷикӣ/русӣ.
  • Веб-панели админ: фармоишҳо, маҳсулот, дашборд, огоҳиҳо, танзимот, содироти CSV, нусхаи БД.
  • Усули оғоз аз тарафи шумо: Docker + дастури қадам-ба-қадам барои Windows (ва дар зарурат Linux/macOS).
  • Ҳуҷҷатҳо: насб барои фармоишгар, тавсифи лоиҳа, дар зарурат — дастур барои барномасоз.
  • Супориш: архив бо лоиҳа ва/ё тасвир дар Docker Hub + файлҳои дастур.
  • Машварати мухтасар оид ба оғози аввал (ҷавоб ба саволҳо то 7 рӯз пас аз супориш).
