Магазин ноутбуков
Telegram-бот + веб-админка

Каталог • Заказы • Оплата • Чеки • AI‑консультант

Единая система для клиентов и администратора. Поддержка русского и таджикского языка. Запуск через Docker на любой ОС.

Клиент: Telegram
Админка: Web (порт 8080)
БД: SQLite
Стек: Python async
Визуальный стиль — тёмный «магазин», без слепящего белого.
1. Цель проекта
B2C продажи в Telegram
  • Сделать удобный канал продаж ноутбуков через Telegram: каталог → заказ → оплата → чек.
  • Сократить ручную работу: статусы заказов, история, экспорт, уведомления и бэкап.
  • Дать администратору веб‑панель для управления товарами и заказами.
2. Актуальность
Таджикистан • RU/TG
  • Telegram — привычный канал коммуникации и продаж, без отдельного сайта.
  • Нужен единый процесс: каталог, реквизиты, чеки и история заказов в одном месте.
  • AI‑подбор помогает клиенту быстрее выбрать устройство по бюджету и требованиям.
  • Администратору нужна централизованная панель вместо таблиц и переписок.
3. Задачи проекта
Каталог • Заказы • Админка
  • Каталог: категории, карточка товара, поиск, избранное, медиа (фото/видео).
  • Оформление заказа (FSM): ФИО, телефон, город, адрес + авто‑номер заказа.
  • Приём оплаты: реквизиты, загрузка фото чека, напоминания клиенту.
  • Веб‑админка: CRUD товаров, заказы со статусами, фильтры, CSV‑экспорт, бэкап БД.
4. Этапы разработки
От прототипа к релизу
  • Проектирование БД и API, базовая асинхронная серверная часть.
  • Каталог и сценарии заказов (FSM), статусы, хранение чеков.
  • Интеграция AI‑консультанта и многоязычность.
  • Админ‑панель: заказы, товары, уведомления, настройки, экспорт и бэкап.
  • Деплой: Docker + инструкция установки для Windows/Linux/macOS.
5. Функциональность
Клиент + Админ
  • Клиент: каталог, избранное, заказ, реквизиты, чек, история.
  • AI‑консультант: подбор по бюджету/задачам, ответы на вопросы, автоязык.
  • Админ: дашборд, управление заказами и товарами, остатки, фильтры, экспорт.
  • Настройки: управление .env, тексты бота (RU/TG), бэкап БД.
6. Технологии
Async • Web • Docker
  • Backend: Python 3.11+, aiogram 3.x, aiohttp, SQLite (aiosqlite).
  • Админка: HTML5 + модульные CSS + модульный JS (ES Modules).
  • Интеграции: Telegram Bot API, OpenRouter API (AI).
  • Деплой: Docker / docker-compose, установка на любой ОС (включая Windows).
7. Ожидаемые результаты
Готово к внедрению
  • Рабочий каталог и заказы для клиентов в Telegram.
  • Прозрачный контроль продаж: статусы, история, чеки, экспорт CSV.
  • Снижение ручной работы администратора за счёт панели и уведомлений.
  • Простая установка и переносимость через Docker.
8. Риски и ограничения
Управляемые риски
  • Зависимость от внешних API (AI) — лимиты и доступность сервиса.
  • Загрузка медиа и чеков — нужны ограничения размера и таймауты.
  • SQLite подходит для малого/среднего объёма; при росте — миграция на PostgreSQL.
  • Безопасность админки: ключ + JWT, при необходимости — ограничение по IP.
9. Планы на будущее
Рост и улучшения
  • Расширение каталога: сравнение, фильтры, отзывы.
  • Авто‑уведомления клиенту о доставке и статусах.
  • Роли админов, аналитика и отчётность.
  • Переход на более мощную БД при росте нагрузки.
10. Участники проекта
Роли и ответственность
  • Backend/API: логика бота, БД, REST API, интеграции (AI).
  • Frontend админки: UI/UX, модули JS/CSS, работа с медиа.
  • Контент/оператор: заполнение каталога, обработка заказов и чеков.
  • Руководитель: постановка задач, приёмка, поддержка и развитие.
